## Как работает синхронизация

### 1. Загрузка данных (getData)

Приложение загружает данные из Google Таблицы:
```
GET https://script.google.com/macros/s/.../exec?mode=jsonp&action=getData&callback=jsonp_callback_xxx
```

Google Apps Script читает данные из таблицы и возвращает JSON.

### 2. Сохранение данных (saveData)

Приложение отправляет данные в Google Таблицу:
```
GET https://script.google.com/macros/s/.../exec?mode=jsonp&action=saveData&data=...
```

**Важно:** Отправляются только рулоны с заполненным `factoryLength`. Пустые рулоны не передаются.

### 3. Логика сохранения

Функция `saveData` в Google Apps Script:
1. Читает ВСЕ существующие данные из таблицы
2. Мержит (объединяет) с новыми данными
3. Перезаписывает таблицу

Это позволяет:
- Добавлять новые рулоны
- Обновлять измерения существующих рулонов
- Сохранять старые данные нетронутыми

### 4. Статусы заказов

Статус автоматически вычисляется в Google Apps Script:

| Status | Условие |
|--------|---------|
| `pending` | Нет измерений |
| `partial` | Есть Factory Length + (Measured Length ИЛИ Shrinkage) |
| `completed` | Есть Measured Length И Shrinkage |

## Настройка Google Apps Script

### Первый раз

1. Откройте [Google Apps Script](https://script.google.com/)
2. Создайте новый проект
3. Скопируйте код из файла `Code.gs` в редактор
4. Нажмите **Deploy** → **New deployment**
5. Выберите тип **Web app**
6. Настройте:
   - Description: v1
   - Execute as: Me
   - Who has access: Anyone (или Anyone with Google account)
7. Нажмите **Deploy**
8. Скопируйте URL веб-приложения

### Обновление кода

1. Откройте проект в Google Apps Script
2. Обновите код (особенно функцию `saveData`)
3. Нажмите **Deploy** → **New deployment**
4. Выберите тип **Web app**
5. Нажмите **Deploy**

**Важно:** При изменении логики сохранения нужно передеплоить!

## Использование

### 1. Сканирование QR-кода

Отсканируйте код формата `НОМЕР_ЗАКАЗА_НОМЕР_РУЛОНА`

Например: `2020_1` (заказ 2020, рулон 1)

### 2. Ручной ввод

Если QR-код не сканируется, введите код вручную.

### 3. Запись измерений

- **Factory Length** - заводская длина (обязательно)
- **Measured Length** - измеренная длина
- **Shrinkage** - усадка в %

### 4. Кнопки синхронизации

- 🔵 **Серая кнопка** (верхняя) - принудительная загрузка из Google Таблицы
- 🔵 **Синяя кнопка** (справа от серой) - обновить страницу
- 🔵 **Серая кнопка** (внизу) - автосинхронизация (каждые 5 сек)

### 5. Добавление заказа

Кнопка "Добавить заказ" - создаёт новый заказ с указанным количеством рулонов.

## Устранение проблем

### Ошибка "JSONP request failed"

- Проверьте URL веб-приложения
- Убедитесь что Google Apps Script задеплоен
- Проверьте консоль браузера (F12 → Console)

### Данные не сохраняются

- Проверьте что ввели Factory Length (обязательно для сохранения)
- Проверьте что Google Apps Script обновлён и передеплоен

### Данные стираются при сохранении

- Обновите Google Apps Script до актуальной версии (функция saveData должна мержить данные)
- Передеплойте веб-приложение

### Локальные данные не синхронизируются

- Используйте https://artteth.github.io/phone-scan/ вместо localhost
