# Спецификация: Приложение инвентаризации рулонов ткани

## 1. Обзор проекта

**Название:** Fabric Roll Inventory Scanner  
**Тип:** Веб-приложение (PWA-ready)  
**Функционал:** Сканирование штрих-кодов/QR-кодов на рулонах ткани, запись измерений (метраж, усадка), управление заказами  
**Целевые пользователи:** Сотрудники склада/производства тканей

---

## 2. Структура данных

### 2.1 Заказ (Order)
```javascript
{
  id: "2020",           // Номер заказа (из штрих-кода)
  totalRolls: 5,        // Всего рулонов в заказе
  rolls: [
    {
      rollNumber: 1,    // Номер рулона (2020_1)
      factoryLength: 50,    // Заводской метраж (м)
      measuredLength: null, // Измеренный метраж (м)
      shrinkage: null,      // Усадка (%)
      status: "pending"     // pending | measured | completed
    },
    // ... остальные рулоны
  ]
}
```

### 2.2 Формат штрих-кода
- **QR-код:** `2020_1`, `2020_2`, `2020_3` и т.д.
- Где `2020` - номер заказа, `_1`, `_2` - номер рулона

---

## 3. UI/UX Спецификация

### 3.1 Страница сканирования (Главная)

**Элементы:**
- Заголовок "Сканер рулонов ткани"
- Кнопка "Сканировать QR-код" (активирует камеру)
- Поле ввода "Или введите вручную" (текстовое поле + кнопка)
- Список последних сканирований (до 5 штук)
- Кнопка перехода к списку всех заказов

**Поведение:**
- При сканировании: парсит код, ищет заказ, открывает форму записи
- При вводе вручную: аналогично сканированию
- Если заказ не найден: создает новый с запросом количества рулонов

### 3.2 Модальное окно записи данных рулона

**Поля:**
- Номер заказа (только чтение)
- Номер рулона (только чтение)
- Заводской метраж (числовое поле, м)
- Измеренный метраж (числовое поле, м) - опционально
- Усадка (числовое поле, %) - опционально

**Кнопки:**
- "Записать" - сохраняет данные
- "Записать частично" - сохраняет только заполненные поля

**Поведение:**
- Автоматически заполняет номер заказа и рулона из сканирования
- Показывает текущий статус рулона
- Позволяет дописывать данные порционно

### 3.3 Страница списка заказов

**Элементы:**
- Таблица/список заказов
- Для каждого заказа: номер, прогресс (X/Y рулонов), статус
- Клик по заказу - детальный вид

### 3.4 Детальный вид заказа

**Элементы:**
- Информация о заказе
- Таблица всех рулонов с их данными
- Возможность редактирования любого рулона

## 5. Техническая реализация

### 5.1 Стек
- Чистый HTML5 + CSS3 + Vanilla JavaScript (ES6+)
- Локальное хранилище (localStorage) для данных
- Библиотека для QR-кодов: qrcode.js (CDN)
- Библиотека для сканирования: html5-qrcode (CDN)

### 5.2 Структура файлов
```
/phone-scan
├── index.html          # Главная страница
├── styles.css          # Стили
├── app.js              # Логика приложения
├── SPEC.md             # Эта спецификация
└── qr-codes/           # Папка с QR-кодами
    ├── 2020_1.png
    ├── 2020_2.png
    └── ...
```

## 6. Дополнительные функции (предложения)

1. **Экспорт данных** - выгрузка в CSV
2. **История изменений** - логирование всех записей
3. **Синхронизация** - потенциальная интеграция с сервером
4. **Оффлайн режим** - PWA с service worker
5. **Статистика** - графики усадки по заказам
6. **Поиск** - фильтрация заказов и рулонов
7. **Звуковые сигналы** - подтверждение сканирования
